# Composer 输入体验

## 理念

Composer 是用户持续组织和修改输入的创作面，不是只在 agent 空闲时开放的一次性提交框。设计优先级应始终偏向方便用户表达，并保护尚未提交的输入；具体能力还会随使用持续积累。

## 当前不变量

1. **输出与输入可以同时发生**：transcript 仍在流式更新时，用户也可能已经开始准备下一条输入。transcript 或运行状态更新不得抢走 composer 焦点、覆盖或清空 draft，也不得阻塞用户继续编辑；agent 是否忙只影响提交后的路由，不影响输入本身。
2. **多行是输入语义的一部分**：用户可能主动换行组织内容。换行、光标位置和未提交 draft 应被完整保留；高度调整、补全、历史/队列召回等交互不能意外归一化或丢失这些内容。

## 演进方式

这不是完整的功能清单。后续发现新的输入便利能力时继续在这里沉淀，并用三个问题判断设计：是否减少用户组织输入时的打断，是否保护用户已经写下的内容与编辑意图，是否避免把 composer 的可用性绑定到 transcript 或 agent 的运行状态。
